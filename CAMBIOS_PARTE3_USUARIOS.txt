- enums/Rol.java


CAMBIOS REALIZADOS RESPECTO AL ORIGINAL
=======================================

1. Cambio de paquete

Antes:

package entities;
package services;
package ui;
package enums;
package exception;

Ahora:

package tpi.progra2.entities;
package tpi.progra2.services;

Esto permite que Usuarios compile junto con Categorias, Productos y Pedidos.


2. Uso de Base comun

Usuario sigue heredando de Base, pero ahora usa:

tpi.progra2.entities.Base

No se debe copiar la Base original del ZIP de usuarios.


3. Uso de Rol comun

Antes usaba:

enums.Rol

Ahora usa:

tpi.progra2.enums.Rol

Motivo:
El enum Rol queda centralizado en el proyecto final.


4. Excepciones unificadas

Antes usaba:

exception.DatoInvalidoException
exception.EntidadNoEncontradaException
exception.MailDuplicadoException

Ahora usa:

tpi.progra2.exceptions.DatoInvalidoException
tpi.progra2.exceptions.EntidadNoEncontradaException
tpi.progra2.exceptions.MailDuplicadoException


5. Metodo importante para pedidos

Se conserva:

buscarUsuarioActivoPorId(Long id)

Motivo:
La parte de pedidos necesita seleccionar un usuario existente y no eliminado.


6. Menu de usuarios

El archivo MenuUsuarios.java original no se dejo como archivo separado.

Sus opciones se integraron en:

src/tpi/progra2/menus/MenuPrincipal.java

Motivo:
El sistema final debe tener un menu principal unico.
